## アプリケーション機能
### 機能一覧
1.店舗検索機能
・GPS検索：Geolocation APIを使用し、現在地周辺（半径300m~3000m）の飲食店を検索
・絞り込み検索: フリーワード、ジャンル、予算、こだわり条件（Wi-Fi、個室、ランチなど）によるフィルタリング。
2.店舗一覧表示機能
・検索結果のカード形式表示。
・ページネーションによる結果の分割表示。
3.店舗詳細表示機能
・店舗の基本情報（営業時間、定休日、予算、設備情報など）の詳細表示。
・Google Maps（経路案内）およびホットペッパーグルメ公式サイトへのリンク連携。
4.お気に入り機能
・気に入った店舗のローカル保存（Local Storage使用）。
・お気に入り店舗の一覧表示および削除機能。

### テーブル定義(ER図)などの設計ドキュメント（ウェブアプリ）
本アプリはバックエンド（DB）を持たないフロントエンド完結のSPAであり、ホットペッパーグルメサーチAPIから取得したデータを一時的に保持、
またはブラウザのLocalStorageにお気に入りIDのみを保存する仕様のため、ER図等のデータベース設計書はありません。


### 開発環境構築手順(ウェブアプリ)
1.リポジトリのクローン
```
git clone https://github.com/HYO-KM/atarino-gourmet
```
2.依存関係のインストール
npm install
```
3.環境変数の設定
・プロジェクトルートに .env ファイルを作成する。
・ホットペッパーグルメサーチAPIのAPIキーを以下の形式で記述する。
```
VITE_HOTPEPPER_API_KEY=あなたのAPIキー
```
4.ローカルサーバーの起動
```
npm run dev
```
5.動作確認
表示されたURLにアクセスする


#### コンセプト
「今いる場所」から、外さない「あたり」のお店をサクッと見つける。 
複雑になり過ぎず、現在地周辺のお店を直感的に探せるグルメ検索アプリです。

#### こだわったポイント
1.アクセスからすぐに検索できるスピード重視の設計
今すぐお店を探したいというユーザーのニーズに応えるため、あえてログイン機能などを廃止し、
検索条件の必須項目も無くしました。
アプリを開いて即座に周辺のお店が見つけられる手軽さとスピードを軸に設計しました。

2.現在の状況に合わせた検索アシスト
ただお店を表示するだけでなく、ユーザーのお店選びをサポートするために
現在地からの距離に加え、キーワード検索機能や予算による絞り込み機能を実装しました。
リアルな利用シーンに即したさまざまな検索が可能です。


### デザイン面でこだわったポイント
1.利用シーンに最適化するレスポンシブデザイン
移動中のモバイルや自宅のPCなど多様な利用シーンを想定し、画面幅に応じてレイアウトが最適化される
デザインにしました。デバイス問わず、常に最適な操作性を提供します。

2.SPAによるストレスフリーな操作性
ページの遷移ごとの読み込みによるストレスを排除するために、SPAとして構築しました。
滑らかな画面遷移と素早い反応を実装し、ネイティブアプリのような心地よさを目指しました。

3.ユーザーへの親切なUI設計
検索中のローディング表示や検索結果が0件の場合のガイダンスなど、システムの状況を可視化しました。
今何が起きているのかをユーザーに伝えることで安心して使えるアプリを目指しました。


### 技術面でアドバイスして欲しいポイント
コンポーネントの分割粒度に関してです。現在は SearchForm, ShopList, ShopDetail と大きく分けていますが、
さらに細かく分けるべき基準やベストプラクティスがあれば知りたいです。


### 自己評価
まず「誰のためのアプリか」という企画段階から着手し、「現在地から自分に合う店舗をすぐに見つけられる」というコンセプトを明確にして開発を進めました。
開発においては、このコンセプトを程よく体現し、設計通りの機能を最後まで実装することができました。自分で決めた要件を形にできたことは、エンジニアとしての自信に繋がっています。
一方で、実際に制作中のアプリを自分で操作してみると、「絞り込み条件はこれで足りるか？」「詳細情報は十分か？」といった、設計段階では見えなかった課題や違和感にも気づくことができました。
この経験から、「アプリは作って終わりではなく、実際に使い込んで改善を繰り返すプロセスも重要」と深く学びました。
今後はこの経験を活かし、ユーザーレビューやフィードバックを積極的に取り入れながら、より使いやすいアプリを制作していくための知識と技術を身につけていきたいです。
